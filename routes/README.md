## 🐾 Branch Strategy

<details markdown="1">
<summary>브랜치 종류 소개</summary>

</details>

<details markdown="1">
<summary>시나리오</summary>

> 1️⃣ **Issue**
> 1. API 나누기

> 2️⃣ **Fork repository**

> 3️⃣ **Pull request**
> 1. reviewer → 3명
> 2. 3명 모두 승인할 때 merge 

> 4️⃣ **merge**
> 1. 팀원 호출
> 2. 리뷰, 피드백, 회의 마친 후
> 3. 다 같이 보는 자리에서 merge
